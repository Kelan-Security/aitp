The Problem

TCP/IP blindness.

Standard transport protocols see bytes, not intentions. When an AI agent connects to your database, legacy firewalls see "Traffic on Port 5432" until the breach is already complete.

LLM Agent A → HTTP/JSON → TCP → Database
// Identity: unknown
// Intent: unknown
// Trust: unknown
The Solution

AITP handshake.

AITP enforces identity and intent before a single application byte is exchanged. Every session is cryptographically bound and AI-evaluated at the wire.

01 HELLO [identity]
02 EXCHANGE [challenge]
03 DECLARE [intent: model_inference]
04 EVAL [gemini: approved]
05 ESTABLISHED // 3.1ms

Protocol Specification

Primitives.

Primitive 1 — Identity

Every node generates a unique Ed25519 keypair. The EntityID is the stable identifier used for ACLs and trust scores.

EntityID = SHA-256(Ed25519_PublicKey)
struct EntityIdentity {
  id: [u8; 32], // SHA-256 hash
  public_key: Vec<u8>,
  metadata: HashMap<String, String>,
}

Primitive 2 — Intent

Clients must explicitly declare what they intend to do before the pipe opens. Unknown or undeclared intents are dropped.

enum IntentCode {
  ModelInference = 0x01,
  DataSync = 0x02,
  ControlSignal = 0x03,
  IdentityEx = 0x04,
  AuditLog = 0x05,
  Heartbeat = 0x06,
  Discovery = 0x07,
}

Primitive 3 — Trust

Trust is calculated dynamically. A session is only granted if the merged score exceeds the threshold.

S_merged = 0.4 × S_rules + 0.6 × S_AI
// Trust Thresholds
Allow: ≥ 128
Monitor: 64 - 127
Deny: < 64

The Vision

Why we are building AITP.

TCP/IP was designed for a world of trusted hosts and static routes. It cannot see the "Intent" of an LLM agent requesting access to a sensitive API.

AITP provides the missing metadata layer. By moving trust evaluation from the application to the transport layer, we eliminate entire classes of injection and lateral movement attacks before they reach your code.
