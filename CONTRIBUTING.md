# Contributing to AITP

Thank you for your interest in AITP! We welcome contributions to make the future of AI transport safer and faster.

## How to Contribute
1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Ensure all tests pass (`make test`).
4. Submit a Pull Request with a clear description of your changes.

## CLA
All contributors must sign a Contributor License Agreement (CLA) before their PRs can be merged.

## Standards
- Follow Rust best practices and `fmt` conventions.
- Include unit tests for all new features.
- Update documentation as needed.
